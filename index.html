// 01 · Conceptos

¿Qué es Multi-Cloud?

Estrategia que utiliza infraestructura de dos o más proveedores de nube pública en lugar de depender de uno solo.

☁ Multi-Cloud

Combina múltiples nubes públicas (AWS + Azure + GCP) para flexibilidad, resiliencia y optimización. Workloads distribuidos exclusivamente entre proveedores públicos.

⬡ Nube Híbrida

Mezcla nubes públicas con privadas u on-premises para control y cumplimiento normativo. Datos sensibles permanecen en infraestructura propia.

// 05 · Desafíos

Los Retos Reales

Haz clic en cada desafío para profundizar en soluciones.

⚡

Latencia

Coordinación inter-nodo agrega ~5ms mínimo. Distancias entre nubes impactan replicación real-time.

Límite síncrona: ~300km
🔒

Seguridad

Superficie de ataque ampliada. Políticas IAM inconsistentes entre AWS, Azure y GCP.

Solución: Zero-Trust
💰

Costos

Egress fees entre nubes ($0.01-0.09/GB). Pricing variado y gastos inesperados.

Solución: FinOps
⚙

Complejidad

Múltiples APIs, herramientas nativas diferentes, skills gaps del equipo técnico.

Solución: Data Fabrics
// 06 · Tecnologías

Bases de Datos Distribuidas

A
AWS

Aurora Global Database

Réplicas de lectura cross-region. Compatible MySQL/PostgreSQL. Un solo master — no es SQL distribuido verdadero.

S
GCP

Cloud Spanner

SQL distribuido global con TrueTime API (relojes atómicos + GPS). Inspiró a CockroachDB.

C
Multi-Cloud

CockroachDB

SQL distribuido cloud-native. Cada nodo lee y escribe. Consenso Raft. Cualquier nube.

T
Open Source

TiDB

HTAP: TiKV (OLTP) + TiFlash (OLAP). Compatible MySQL. Replicación real-time entre stores.
